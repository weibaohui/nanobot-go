## 1. 设计目标

将大模型的思考过程（工具调用、助手回复等）实时发送到 channel，提升用户体验。

---

## 2. 架构设计

### 2.1 组件关系

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   LLM / Tools   │────▶│   HookManager    │────▶│ Dispatcher      │
└─────────────────┘     └──────────────────┘     └────────┬────────┘
                                                          │
                                                          ▼
                                                ┌─────────────────────┐
                                                │ ThinkingProcess     │
                                                │ Observer            │
                                                └──────────┬──────────┘
                                                           │
                                                           ▼
                                                ┌─────────────────────┐
                                                │   MessageBus        │
                                                │   (Outbound)        │
                                                └──────────┬──────────┘
                                                           │
                                                           ▼
                                                ┌─────────────────────┐
                                                │   Matrix / Other    │
                                                │   Channel           │
                                                └─────────────────────┘
```

### 2.2 会话信息传递方案

通过 context 传递 sessionKey 和 channel：

```
loop.processMessage()
    ↓ 注入 sessionKey/channel 到 context
    ↓ ctx = trace.WithSessionInfo(ctx, sessionKey, channel)
    ↓
MasterAgent.Process(ctx, msg)
    ↓
LLM.Generate(ctx, ...)
    ↓
EinoCallbackBridge.onStart(ctx, ...)
    ↓ 从 context 获取 sessionKey/channel
    ↓ sessionKey := trace.GetSessionKey(ctx)
    ↓ channel := trace.GetChannel(ctx)
    ↓
Dispatcher.Dispatch(ctx, event, channel, sessionKey)
    ↓ 补充 context 中的会话信息
    ↓
ThinkingProcessObserver.OnEvent(ctx, event)
    ↓ 从 context 获取 sessionKey
    ↓ 从缓存获取 chatID
    ↓
MessageBus.PublishOutbound(msg)
```

## 3. 详细设计

### 3.1 配置结构

```go
// ThinkingProcessConfig 思考过程配置
type ThinkingProcessConfig struct {
    Enabled bool     `json:"enabled"` // 是否启用
    Events  []string `json:"events"`  // 要监听的事件类型
}
```

默认监听事件：
- `tool_used` - 工具调用
- `tool_completed` - 工具完成
- `llm_call_end` - LLM 响应

### 3.2 Context 扩展

在 `agent/hooks/trace/context.go` 中添加：

```go
// SessionKeyKey 是 context 中存储 SessionKey 的 key
type SessionKeyKey struct{}

// ChannelKey 是 context 中存储 Channel 的 key
type ChannelKey struct{}

// WithSessionKey 将 SessionKey 注入到 context 中
func WithSessionKey(ctx context.Context, sessionKey string) context.Context

// WithChannel 将 Channel 注入到 context 中
func WithChannel(ctx context.Context, channel string) context.Context

// WithSessionInfo 将会话信息注入到 context 中
func WithSessionInfo(ctx context.Context, sessionKey, channel string) context.Context

// GetSessionKey 从 context 中获取 SessionKey
func GetSessionKey(ctx context.Context) string

// GetChannel 从 context 中获取 Channel
func GetChannel(ctx context.Context) string
```

### 3.3 Observer 实现

```go
// ThinkingProcessObserver 思考过程观察器
type ThinkingProcessObserver struct {
    *observer.BaseObserver
    config     *config.ThinkingProcessConfig
    messageBus *bus.MessageBus
    logger     *zap.Logger

    // 会话信息缓存（sessionKey -> chatID）
    sessionCache map[string]sessionInfo
    mu           sync.RWMutex
}
```

### 3.4 消息格式

#### 工具调用 (tool_used)

```markdown
🔧 **调用工具**: `{tool_name}`
```
{arguments}
```
```

#### 工具完成 (tool_completed)

```markdown
✅ **工具完成**: `{tool_name}`
```
{response}
```
```

#### LLM 响应 (llm_call_end)

如果有工具调用：
```markdown
🤖 **准备调用工具**: tool1, tool2
```

如果只有文本响应：
```markdown
{response_content} (截断到500字符)
```

## 4. 关键问题解决方案

### 4.1 如何获取 ChatID？

**方案**：
1. 监听 `MessageReceivedEvent`，缓存 sessionKey -> (chatID, channel) 映射
2. 通过 context 传递 sessionKey
3. 在 Observer 中从 context 获取 sessionKey，查找对应的 chatID

### 4.2 如何避免重复发送最终响应？

**方案**：
- 在 `llm_call_end` 时只发送部分内容（截断到500字符）
- 如果有工具调用，只显示工具调用信息
