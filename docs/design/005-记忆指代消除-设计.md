## 1. 设计目标

### 1.1 要解决的问题

当前记忆模块生成对话总结时，仅基于单条对话内容进行提炼，存在以下问题：

1. **代词指代不明**：用户说"它怎么样"、"这个方案"，脱离上下文无法理解具体指代
2. **省略主语**：连续对话中经常省略主语，单看一条记录无法确定讨论对象
3. **时间/地点指代**："昨天说的"、"上次会议"，缺乏时间锚点
4. **总结质量受限**：LLM 仅基于单条对话总结，信息不完整

### 1.2 设计目标

构建指代消除（Coreference Resolution）功能模块，在写入流水记忆前：
1. 获取当前对话的上下文（前后两小时内的同 session 对话）
2. 使用 LLM 或规则引擎进行指代消解
3. 将模糊表述转换为明确表述
4. 生成高质量的记忆内容

### 3.2 上下文查询服务设计

#### 3.2.1 查询逻辑

```go
type contextQueryService struct {
    conversationRepo conversation.Repository
}

func (s *contextQueryService) GetContextForResolution(
    ctx context.Context,
    traceID string,
    sessionKey string,
    timestamp time.Time,
    timeWindow time.Duration,
    maxMessages int,
) (*ContextResult, error) {
    // 计算时间范围
    startTime := timestamp.Add(-timeWindow)
    endTime := timestamp.Add(timeWindow)

    // 查询前文（限制数量）
    previousOpts := &models.QueryOptions{
        OrderBy: "timestamp",
        Order:   "DESC",  // 倒序，最近的在前
        Limit:   maxMessages / 2,
    }
    previous, err := s.conversationRepo.FindByTimeRange(
        ctx, startTime, timestamp, previousOpts,
    )
    // 反转回正序
    reverse(previous)

    // 查询当前对话
    current, err := s.conversationRepo.FindByTraceID(ctx, traceID)

    // 查询后文（限制数量）
    followingOpts := &models.QueryOptions{
        OrderBy: "timestamp",
        Order:   "ASC",
        Limit:   maxMessages / 2,
    }
    following, err := s.conversationRepo.FindByTimeRange(
        ctx, timestamp, endTime, followingOpts,
    )

    return &ContextResult{
        CurrentConversation: convertToMessages(current, 0),
        PreviousMessages:    convertToMessages(previous, -len(previous)),
        FollowingMessages:   convertToMessages(following, 1),
        TotalMessages:       len(previous) + len(current) + len(following),
        TimeRangeStart:      startTime,
        TimeRangeEnd:        endTime,
    }, nil
}
```

#### 3.2.2 性能优化

- **索引优化**：确保 conversation_records 表有 (session_key, timestamp) 复合索引
- **缓存机制**：短时间内相同 trace_id 的查询结果缓存 5 分钟
- **限流控制**：单 session 并发查询限制

#### 3.3.2 轻量模式（规则引擎）

```go
type RuleBasedResolver struct {
    rules []ResolutionRule
}

func (r *RuleBasedResolver) Resolve(
    ctx context.Context,
    input *ResolutionInput,
    context *ContextResult,
    mode ResolutionMode,
) (*ResolutionOutput, error) {
    text := joinMessages(input.Messages)
    replacements := make([]Replacement, 0)

    for _, rule := range r.rules {
        matches := rule.Pattern.FindAllStringIndex(text, -1)
        for _, match := range matches {
            resolved, ok := rule.Extractor(text[match[0]:match[1]], context)
            if ok {
                replacements = append(replacements, Replacement{
                    Original:   text[match[0]:match[1]],
                    Resolved:   resolved,
                    Position:   match[0],
                    Type:       rule.Type,
                    Confidence: 0.8,
                })
            }
        }
    }

    // 应用替换（从后往前，避免位置偏移）
    resolvedText := applyReplacements(text, replacements)

    return &ResolutionOutput{
        OriginalText:   text,
        ResolvedText:   resolvedText,
        Replacements:   replacements,
        Confidence:     calculateConfidence(replacements),
        ModeUsed:       ModeLight,
        ProcessingTime: time.Since(start),
    }, nil
}
```

#### 3.3.3 深度模式（LLM）

```go
type LLMBasedResolver struct {
    client LLMClient
    prompt string
}

func (r *LLMBasedResolver) Resolve(
    ctx context.Context,
    input *ResolutionInput,
    context *ContextResult,
    mode ResolutionMode,
) (*ResolutionOutput, error) {
    // 构建提示词
    prompt := r.buildPrompt(input, context)

    // 调用 LLM
    response, err := r.client.Complete(ctx, prompt, "")
    if err != nil {
        return nil, err
    }

    // 解析响应
    return r.parseResponse(response), nil
}

func (r *LLMBasedResolver) buildPrompt(input *ResolutionInput, context *ContextResult) string {
    var sb strings.Builder

    sb.WriteString("你是一位专业的指代消除专家...\n\n")
    sb.WriteString("## 上下文对话\n")
    for _, msg := range context.PreviousMessages {
        sb.WriteString(fmt.Sprintf("[%s] %s: %s\n",
            msg.Timestamp.Format("15:04"), msg.Role, msg.Content))
    }

    sb.WriteString("\n## 目标对话（需要消除指代）\n")
    for _, msg := range input.Messages {
        sb.WriteString(fmt.Sprintf("[%s] %s: %s\n",
            msg.Timestamp.Format("15:04"), msg.Role, msg.Content))
    }

    sb.WriteString("\n## 输出要求\n")
    sb.WriteString("请按JSON格式返回...")

    return sb.String()
}
```

#### 3.3.4 自动模式

```go
type AutoResolver struct {
    light CoreferenceResolver
    deep  CoreferenceResolver
}

func (r *AutoResolver) Resolve(
    ctx context.Context,
    input *ResolutionInput,
    context *ContextResult,
    mode ResolutionMode,
) (*ResolutionOutput, error) {
    // 判断复杂度
    complexity := r.assessComplexity(input, context)

    if complexity < 0.3 {
        // 简单场景：使用轻量模式
        return r.light.Resolve(ctx, input, context, ModeLight)
    }

    // 复杂场景：使用深度模式
    return r.deep.Resolve(ctx, input, context, ModeDeep)
}

func (r *AutoResolver) assessComplexity(input *ResolutionInput, context *ContextResult) float64 {
    score := 0.0

    // 1. 代词数量
    pronouns := countPronouns(input)
    score += float64(pronouns) * 0.1

    // 2. 是否有省略主语
    if hasEllipsis(input) {
        score += 0.2
    }

    // 3. 上下文长度
    if len(context.PreviousMessages) > 10 {
        score += 0.2
    }

    // 4. 跨轮指代（前文提到多个实体）
    entities := extractEntities(context.PreviousMessages)
    if len(entities) > 2 {
        score += 0.3
    }

    return min(score, 1.0)
}
```

#### 3.4.1 MemorySummarizer 改造

```go
type MemorySummarizer struct {
    client          LLMClient
    conversationSvc ConversationService       // 新增
    resolver        CoreferenceResolver       // 新增
    config          CoreferenceConfig         // 新增
    logger          *zap.Logger
}

func (s *MemorySummarizer) SummarizeConversation(
    ctx context.Context,
    messages []Message,
    traceID string,           // 新增
    sessionKey string,        // 新增
    timestamp time.Time,      // 新增
) (*ConversationSummary, error) {

    // 如果启用指代消除
    if s.config.Enabled {
        resolved, err := s.performResolution(ctx, messages, traceID, sessionKey, timestamp)
        if err == nil && resolved.Confidence >= s.config.MinConfidence {
            // 使用消除后的内容生成总结
            return s.summarizeWithResolvedText(ctx, resolved)
        }
        // 降级：记录日志，使用原始内容
        s.logger.Warn("指代消除失败或置信度低，使用原始内容",
            zap.Error(err),
            zap.Float64("confidence", resolved.Confidence),
        )
    }

    // 原流程
    return s.summarizeOriginal(ctx, messages)
}

func (s *MemorySummarizer) performResolution(
    ctx context.Context,
    messages []Message,
    traceID string,
    sessionKey string,
    timestamp time.Time,
) (*ResolutionOutput, error) {
    // 1. 获取上下文
    context, err := s.conversationSvc.GetContextForResolution(
        ctx, traceID, sessionKey, timestamp,
        s.config.TimeWindow, s.config.MaxContextMessages,
    )
    if err != nil {
        return nil, err
    }

    // 2. 执行消除
    input := &ResolutionInput{
        TraceID:    traceID,
        SessionKey: sessionKey,
        Messages:   messages,
    }

    return s.resolver.Resolve(ctx, input, context, s.config.Mode)
}
```

## 5. 技术决策

### 5.1 为什么采用双模式策略？

| 模式 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| 轻量模式 | 快（<10ms）、成本低 | 能力有限、规则维护 | 简单代词、实时性要求高 |
| 深度模式 | 能力强、覆盖复杂场景 | 慢（1-2s）、成本高 | 复杂指代、跨多轮对话 |
| 自动模式 | 兼顾效率和效果 | 实现复杂 | 通用场景 |

### 5.2 为什么需要保留原始内容？

1. **可审计**：便于追溯消除是否正确
2. **可回滚**：消除错误时可恢复原文
3. **可学习**：用于优化消除模型
4. **调试**：便于排查问题

### 5.3 时间窗口为什么设为 2 小时？

- **足够长**：覆盖大多数连续对话场景
- **不会太长**：避免引入过多无关上下文
- **可配置**：实际需求中可调整

## 6. 安全考虑

### 6.1 数据隔离

- 上下文查询严格限制同 session_key
- 不跨用户/跨 session 查询
- 敏感信息在消除前过滤

### 6.2 隐私保护

- 日志中不记录完整对话内容
- 定期清理消除日志（30天）
- 支持用户关闭指代消除功能

## 7. 性能考虑

### 7.1 优化策略

| 策略 | 实现方式 | 效果 |
|------|----------|------|
| 索引优化 | (session_key, timestamp) 复合索引 | 查询 < 100ms |
| 结果缓存 | 相同 trace_id 缓存 5 分钟 | 减少重复查询 |
| 轻量优先 | 简单场景用规则引擎 | 90% 场景 < 10ms |
| 超时降级 | LLM 10s 超时后转规则引擎 | 保证可用性 |
| 并发限制 | 单 session 限流 | 防止资源耗尽 |

### 7.2 性能指标

| 场景 | 目标耗时 |
|------|----------|
| 上下文查询 | < 100ms |
| 轻量模式消除 | < 10ms |
| 深度模式消除 | < 2s |
| 全流程（轻量）| < 200ms |
| 全流程（深度）| < 3s |

| 消除失败 | 使用原始内容继续 |
| 置信度过低 | 使用原始内容，记录日志 |
| LLM 超时 | 降级为轻量模式 |
| 轻量模式也失败 | 使用原始内容 |
