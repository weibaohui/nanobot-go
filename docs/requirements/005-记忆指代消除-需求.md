## 1. 需求背景

### 1.1 问题描述

当前记忆模块在生成对话总结时，直接基于单条对话内容进行提炼，存在以下问题：

1. **代词指代不明**：用户说"它怎么样"、"这个方案"、"那位客户"，脱离上下文无法理解具体指代
2. **省略主语**：连续对话中经常省略主语，如"价格多少？""质量如何？"，单看一条记录无法确定讨论对象
3. **时间/地点指代**："昨天说的"、"上次会议"、"那个项目"，缺乏时间锚点
4. **总结质量受限**：LLM 仅基于单条对话总结，信息不完整，导致记忆内容模糊

### 1.2 示例场景

**场景一：代词指代**
```
用户: 这个方案的成本太高了
AI: 是的，可以考虑优化
用户: 它的交付周期也是问题
        ↑ "它"指代什么？
```

**场景二：省略主语**
```
用户: Qwen 2.5 模型怎么样？
AI: 性能很好，开源友好
用户: 价格呢？
        ↑ 问的是 Qwen 2.5 的价格
用户: 支持多少上下文长度？
        ↑ 问的还是 Qwen 2.5
```

**场景三：跨对话引用**
```
用户: 帮我查一下北京天气
AI: 北京今天晴天，25度
用户: 那上海呢？
        ↑ "那"表示对比，"上海"对比"北京"
用户: 明天呢？
        ↑ 问的是上海明天天气
```

### 1.3 需求目标

通过引入**指代消除**（Coreference Resolution）功能，在写入流水记忆前：
1. 获取当前对话的上下文（前后两小时内的同 session 对话）
2. 使用 LLM 或规则引擎进行指代消解
3. 将模糊表述转换为明确表述
4. 生成高质量的记忆内容

## 2. 核心概念

### 2.1 指代消除定义

指代消除（Coreference Resolution）是指识别文本中代词、名词短语与实际所指实体之间关系的过程。

**类型分类**：

| 类型 | 示例 | 消除后 |
|------|------|--------|
| 人称代词 | 它/他/她/他们 | 具体人名/物体名 |
| 指示代词 | 这个/那个/这些 | 具体事物 |
| 时间指代 | 昨天/上次/之前 | 具体日期/时间点 |
| 地点指代 | 这里/那里 | 具体地点 |
| 省略主语 | （省略）怎么样？ | 补全主语 |

### 2.2 上下文窗口

指代消除需要依赖上下文信息，上下文窗口定义为：

| 维度 | 范围 | 说明 |
|------|------|------|
| 时间窗口 | ±2小时 | 以当前对话开始时间为锚点 |
| Session 范围 | 同 session_key | 确保是同一会话 |
| 消息数量 | 最多 20 条 | 避免上下文过长 |

### 2.3 处理时机

```
对话完成事件触发
    │
    ▼
┌─────────────────┐
│ 1. 获取上下文    │ ◄── 查询前后2小时同 session 对话
│    (对话记录服务)│
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 2. 指代消除      │ ◄── 使用 LLM 进行指代消解
│    (本模块)      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 3. 生成总结      │ ◄── 基于消除后的内容生成总结
│    (原流程)      │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 4. 写入流水记忆  │ ◄── 写入高质量记忆
│    (原流程)      │
└─────────────────┘
```

## 3. 功能需求

### 3.1 上下文获取服务

#### 3.1.1 功能描述

提供专用接口获取指代消除所需的上下文对话记录。

#### 3.1.2 接口定义

```go
// ContextQueryService 上下文查询服务
type ContextQueryService interface {
    // GetContextForResolution 获取指代消除所需的上下文
    // traceID: 当前对话的 trace_id
    // sessionKey: 当前对话的 session_key
    // timestamp: 当前对话的时间锚点
    // timeWindow: 时间窗口（前后各多少分钟）
    // maxMessages: 最大消息数量限制
    GetContextForResolution(
        ctx context.Context,
        traceID string,
        sessionKey string,
        timestamp time.Time,
        timeWindow time.Duration,
        maxMessages int,
    ) (*ContextResult, error)
}

// ContextResult 上下文查询结果
type ContextResult struct {
    CurrentConversation []ConversationMessage   // 当前对话（指代消除目标）
    PreviousMessages    []ConversationMessage   // 前文对话（时间窗口内）
    FollowingMessages   []ConversationMessage   // 后文对话（时间窗口内）
    TotalMessages       int                     // 总消息数
    TimeRangeStart      time.Time               // 时间范围开始
    TimeRangeEnd        time.Time               // 时间范围结束
}

// ConversationMessage 对话消息
type ConversationMessage struct {
    ID        uint      `json:"id"`
    TraceID   string    `json:"trace_id"`
    Role      string    `json:"role"`      // user/assistant/system
    Content   string    `json:"content"`
    Timestamp time.Time `json:"timestamp"`
    Distance  int       `json:"distance"`  // 距离当前对话的消息数（负数为前，正数为后）
}
```

#### 3.1.3 查询逻辑

```sql
-- 查询前文（时间窗口内，早于当前对话）
SELECT * FROM conversation_records
WHERE session_key = ?
  AND timestamp >= ?  -- 时间窗口开始
  AND timestamp < ?   -- 当前对话时间
ORDER BY timestamp DESC
LIMIT ?

-- 查询后文（时间窗口内，晚于当前对话）
SELECT * FROM conversation_records
WHERE session_key = ?
  AND timestamp > ?   -- 当前对话时间
  AND timestamp <= ?  -- 时间窗口结束
ORDER BY timestamp ASC
LIMIT ?
```

#### 3.1.4 验收标准

- [ ] 能根据 trace_id 获取当前对话完整消息
- [ ] 能获取前后2小时内的同 session 对话
- [ ] 支持限制返回消息数量（默认20条）
- [ ] 消息按时间顺序排列
- [ ] 支持时间窗口自定义配置

### 3.2 指代消除引擎

#### 3.2.1 功能描述

基于上下文对话，使用 LLM 进行指代消解，将模糊表述转换为明确表述。

#### 3.2.2 处理策略

采用**双模式策略**：

| 模式 | 适用场景 | 实现方式 |
|------|----------|----------|
| 轻量模式 | 简单代词（它/这个/那个） | 规则引擎 + 简单替换 |
| 深度模式 | 复杂指代、跨多轮对话 | LLM 指代消解 |

#### 3.2.3 接口定义

```go
// CoreferenceResolver 指代消除器接口
type CoreferenceResolver interface {
    // Resolve 执行指代消除
    // input: 待消除的原始对话
    // context: 上下文对话记录
    // mode: 处理模式（light/deep/auto）
    Resolve(
        ctx context.Context,
        input *ResolutionInput,
        context *ContextResult,
        mode ResolutionMode,
    ) (*ResolutionOutput, error)

    // BatchResolve 批量处理多个对话
    BatchResolve(
        ctx context.Context,
        inputs []*ResolutionInput,
        contexts []*ContextResult,
        mode ResolutionMode,
    ) ([]*ResolutionOutput, error)
}

// ResolutionMode 处理模式
type ResolutionMode string

const (
    ModeLight ResolutionMode = "light"  // 轻量模式（规则引擎）
    ModeDeep  ResolutionMode = "deep"   // 深度模式（LLM）
    ModeAuto  ResolutionMode = "auto"   // 自动选择
)

// ResolutionInput 指代消除输入
type ResolutionInput struct {
    TraceID    string
    SessionKey string
    Messages   []Message  // 当前对话的消息列表
}

// ResolutionOutput 指代消除输出
type ResolutionOutput struct {
    OriginalText      string            // 原始文本（拼接后）
    ResolvedText      string            // 消除后的文本
    Replacements      []Replacement     // 替换记录
    Confidence        float64           // 置信度（0-1）
    ModeUsed          ResolutionMode    // 实际使用的模式
    ProcessingTime    time.Duration     // 处理耗时
}

// Replacement 替换记录
type Replacement struct {
    Original    string  // 原始表述
    Resolved    string  // 消除后表述
    Position    int     // 在文本中的位置
    Type        string  // 指代类型（pronoun/ellipsis/temporal/spatial）
    Confidence  float64 // 置信度
}
```

#### 3.2.4 LLM 提示词设计（深度模式）

```
你是一位专业的指代消除专家。你的任务是将对话中的模糊指代表述转换为明确的完整表述。

## 任务说明

输入包含两部分：
1. **上下文对话**：当前对话前后相关的对话记录
2. **目标对话**：需要进行指代消除的当前对话

你需要：
1. 识别目标对话中的所有指代（代词、省略、时间/地点指代）
2. 根据上下文确定每个指代的具体含义
3. 将指代表述替换为明确表述
4. 保持原意不变，只消除歧义

## 指代类型

1. **人称/物主代词**：它、他、她、他们、这个、那个、这些、那些
2. **省略主语**：句子缺少主语，需要根据前文补全
3. **时间指代**：昨天、上次、之前、刚才、后来
4. **地点指代**：这里、那里、这边、那边
5. **事件指代**：那件事、那个问题、那个方案

## 输出格式

请按以下JSON格式返回：

{
  "resolved_text": "消除指代后的完整对话文本",
  "replacements": [
    {
      "original": "它",
      "resolved": "该AI模型",
      "position": 15,
      "type": "pronoun",
      "confidence": 0.95
    }
  ],
  "confidence": 0.92
}

## 示例

**上下文**：
[09:00] 用户: 我想了解一下 Claude 3 模型
[09:01] AI: Claude 3 是 Anthropic 发布的最新模型，性能很强

**目标对话**：
[09:05] 用户: 它的价格是多少？

**正确输出**：
{
  "resolved_text": "Claude 3 模型的价格是多少？",
  "replacements": [
    {
      "original": "它",
      "resolved": "Claude 3 模型",
      "position": 0,
      "type": "pronoun",
      "confidence": 0.98
    }
  ],
  "confidence": 0.98
}

## 注意事项

1. 只消除有明确上下文的指代，不确定的保持原样
2. 时间指代要转换为绝对时间（如"昨天"→"2026-03-01"）
3. 保持对话的自然性，不要过度改写
4. 如果上下文不足以消除指代，保持原文并标记低置信度

## 实际输入

**上下文**：
{{CONTEXT}}

**目标对话**：
{{TARGET}}
```

#### 3.2.5 规则引擎设计（轻量模式）

```go
// RuleBasedResolver 基于规则的指代消除器
type RuleBasedResolver struct {
    rules []ResolutionRule
}

// ResolutionRule 消除规则
type ResolutionRule struct {
    Name        string
    Pattern     *regexp.Regexp
    Extractor   func(match []string, context *ContextResult) (string, bool)
    Priority    int
}

// 预定义规则示例
var DefaultRules = []ResolutionRule{
    {
        Name:    "人称代词-它",
        Pattern: regexp.MustCompile(`^它`),
        Extractor: func(match []string, ctx *ContextResult) (string, bool) {
            // 从前文找最近的名词实体
            return extractNearestNoun(ctx.PreviousMessages), true
        },
        Priority: 1,
    },
    {
        Name:    "指示代词-这个/那个",
        Pattern: regexp.MustCompile(`^[这个|那个]+`),
        Extractor: extractNearestNoun,
        Priority: 2,
    },
    {
        Name:    "时间指代-昨天",
        Pattern: regexp.MustCompile(`昨天`),
        Extractor: func(match []string, ctx *ContextResult) (string, bool) {
            yesterday := ctx.TimeRangeStart.AddDate(0, 0, -1)
            return yesterday.Format("2006-01-02"), true
        },
        Priority: 3,
    },
    // ... 更多规则
}
```

## 5. 非功能需求

### 5.1 可靠性

| 需求项 | 要求 |
|--------|------|
| 降级策略 | LLM 失败时自动降级为轻量模式 |
| 超时处理 | 深度模式设置 10s 超时，超时后使用轻量模式 |
| 失败回退 | 指代消除失败时，使用原始内容继续流程 |
| 幂等性 | 相同输入产生相同输出，不重复消除 |

### 5.2 可观测性

| 需求项 | 要求 |
|--------|------|
| 日志记录 | 记录每次消除的输入、输出、耗时、置信度 |
| 指标统计 | 统计消除成功率、平均耗时、模式分布 |
| 审计追踪 | 保留消除前后文本对比，支持人工审核 |

### 5.3 安全与隐私

| 需求项 | 要求 |
|--------|------|
| 数据隔离 | 上下文查询仅限同 session |
| 敏感信息 | 消除过程中识别并过滤敏感信息 |
| 数据 retention | 日志保留 30 天，支持自动清理 |

## 6. 接口与集成

### 6.1 与记忆模块集成

```go
// MemorySummarizer 扩展
type MemorySummarizer struct {
    // ... 原有字段 ...
    contextService  ContextQueryService       // 新增：上下文服务
    resolver        CoreferenceResolver       // 新增：指代消除器
    config          CoreferenceConfig         // 新增：配置
}

// SummarizeConversation 改造
func (s *MemorySummarizer) SummarizeConversation(
    ctx context.Context,
    messages []Message,
    traceID string,
    sessionKey string,
    timestamp time.Time,
) (*ConversationSummary, error) {
    // 1. 如果启用指代消除
    if s.config.Enabled {
        // 2. 获取上下文
        context, err := s.contextService.GetContextForResolution(
            ctx, traceID, sessionKey, timestamp,
            s.config.TimeWindow, s.config.MaxContextMessages,
        )
        if err != nil {
            // 降级：继续原流程
            return s.summarizeWithoutResolution(messages)
        }

        // 3. 执行指代消除
        resolved, err := s.resolver.Resolve(ctx, &ResolutionInput{
            Messages: messages,
        }, context, s.config.Mode)
        if err != nil || resolved.Confidence < s.config.MinConfidence {
            // 降级：使用原始内容
            return s.summarizeWithoutResolution(messages)
        }

        // 4. 基于消除后的内容生成总结
        return s.summarizeWithResolvedText(resolved.ResolvedText, resolved)
    }

    // 原流程
    return s.summarizeWithoutResolution(messages)
}
```
