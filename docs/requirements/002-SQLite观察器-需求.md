## 1. 功能概述

新增一个 SQLite 观察器，将所有渠道的所有事件信息存储到 SQLite 数据库中，便于后续查询和分析。

## 2. 功能需求

### 2.1 核心功能

- 监听所有事件类型
- 将事件数据存储到 SQLite 数据库
- 使用纯 Go 实现的 SQLite 驱动（`modernc.org/sqlite`），无需 CGO

### 2.2 数据存储

- 数据库文件存储在配置的 dataDir 目录下
- 文件名：`events.db`
- 表结构包含：
  - 事件基本信息（trace_id, span_id, event_type, timestamp 等）
  - 渠道信息（channel, session_key, sender_id, chat_id）
  - 完整事件数据（JSON 格式）

### 2.3 接口要求

- 实现 `Observer` 接口
- 支持过滤器配置
- 提供 `Close()` 方法用于关闭数据库连接

## 3. 非功能需求

### 3.1 性能

- 支持并发写入
- 使用索引优化查询性能

### 3.2 可靠性

- 数据库连接池管理
- 错误处理和日志记录

## 4. 约束条件

- 不引入 CGO 依赖
- 参考现有 `SessionObserver` 的实现模式
