## 1. 实现概述

实现了思考过程观察器（ThinkingProcessObserver），用于将大模型的思考过程（工具调用、助手回复等）实时发送到 channel，提升用户体验。

### 1.1 核心功能

- ✅ 配置开关控制是否启用
- ✅ 可配置监听的事件类型
- ✅ 通过 context 传递会话信息
- ✅ 实时发送工具调用和 LLM 响应

---

## 2. 实现细节

### 2.1 会话信息传递方案

采用通过 context 传递 sessionKey 和 channel 的方案：

1. **loop.go**：在 `processMessage` 中注入会话信息到 context
   ```go
   ctx = trace.WithSessionInfo(ctx, sessionKey, msg.Channel)
   ```

2. **eino_bridge.go**：从 context 获取会话信息，传递给 Dispatch
   ```go
   sessionKey := trace.GetSessionKey(ctx)
   channel := trace.GetChannel(ctx)
   cb.dispatcher.Dispatch(ctx, event, channel, sessionKey)
   ```

3. **dispatcher.go**：分发时补充 context 中的会话信息
   ```go
   if existingSessionKey == "" && sessionKey != "" {
       ctx = trace.WithSessionKey(ctx, sessionKey)
   }
   ```

4. **thinking_process.go**：从 context 获取 sessionKey，查找对应的 chatID

### 2.2 消息格式

| 事件类型 | 格式 |
|---------|------|
| tool_used | `🔧 **调用工具**: {tool_name}` |
| tool_completed | `✅ **工具完成**: {tool_name}` |
| llm_call_end | 响应内容（截断到500字符）或工具调用列表 |

## 6. 已知限制

1. **ChatID 获取**：依赖 MessageReceivedEvent 的缓存，如果会话信息过期（10分钟），可能无法发送
2. **消息长度**：LLM 响应截断到 500 字符
3. **流式响应**：当前不支持，后续可通过 StreamChunk 扩展
